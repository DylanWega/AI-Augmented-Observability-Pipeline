# AI-Augmented-Observability-Pipeline

Overview
This pipeline automates the "Detection → Diagnosis → Communication" loop for web services. Instead of receiving a vague "Site Down" alert, teams receive a summarized diagnostic report with actionable resolution steps generated by AI.
The Stack
Orchestration: n8n (Scheduled Cron Trigger)
AI Agent: Gemini 2.0 Flash (Issue Summarization & Logic)
Test Environment: Mockoon (Local API Simulation)
Connectivity: ngrok (Secure Tunneling for local testing)
Communication: Slack (Automated Bot Alerts)
Workflow Logic
Scheduled Health Check: An n8n Cron Trigger pings the target service every 60 seconds.
Target Service (Mockoon): Simulated environment returns status codes (e.g., 500 Internal Server Error) for testing.
Error Execution: If an error is detected, the n8n execution logic captures the full error payload.
AI Analysis: Gemini 2.0 Flash analyzes the error message, identifies the root cause, and generates resolution steps.
Smart Alerting: A Slack App posts a formatted message to a dedicated channel with the summary and "Next Steps."
Setup Instructions
n8n: Import the provided workflow JSON.
Slack: Create a Slack App with chat:write scopes and invite it to your alerts channel.
Gemini: Obtain an API Key from Google AI Studio.
Local Testing:
Start a mock server in Mockoon.
Run ngrok http 3000 (or your mock port) to expose the service to n8n.
